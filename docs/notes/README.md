# docs/notes

## Назначение

Журнал обсуждений и **решений** (ADR-lite): зачем выбрали тот или иной подход,
какие были альтернативы, что отклонили. В отличие от остальных документов в
`docs/` (которые описывают «как есть»), заметки фиксируют «почему так» и историю
выбора.

**Правило:** одна заметка — одно направление/решение. Нумерация по порядку
(`NNNN-slug.md`). Принятое решение отражается в основных доках (`architecture.md`,
`ingestion.md`, …); заметка остаётся как обоснование.

## Формат заметки

`Статус` · `Контекст` · `Решения` (выбор + обоснование + отклонённые варианты) ·
`Следствия` · `Открытые вопросы`. Язык — русский.
