# 0004 — Извлечение, паттерны и тест-корпус

- **Статус:** принято (дизайн); реализация ждёт
- **Дата:** 2026-06-21
- **Связано:** [../extraction.md](../extraction.md),
  [../output-mapping.md](../output-mapping.md), [../conventions.md](../conventions.md)

## Контекст

Доводка извлечения на реальном корпусе и тестовое покрытие. Анализ показал, что
конкретный источник базовый, а богатый корпус — в эталоне (185 масок); под
авторитетное 4-вариантное правило ≈20% строк эталона меняют классификацию.

## Решения

| # | Вопрос | Выбор | Обоснование | Отклонено |
|---|---|---|---|---|
| 1 | Покрытие типов сейчас | **Сетевые + .onion + telegram** | Есть в корпусе; email/IPv6 в данных нет (YAGNI), добавим конфигом | Сразу email+IPv6; только текущие (мис-классификация onion/PSL) |
| 2 | Формат тест-корпуса | **Case-table + golden** | Юнит-точность по бакетам + сквозная проверка | Только case-table; только golden |
| 3 | Изоляция тестов | **Тесты не опираются на артефакты проекта** | Смена источника/поля/конфига не должна ронять тесты; проверяем логику | Тесты против `dataframe/`/`source/`/прод-конфига |
| 4 | Оракул классификации | **Курируемый корпус под правило** | Ручной эталон неполн (нет вариантов 2/4) и местами неточен | Эталон как оракул |

## Нормализация (зафиксировано)

- Хвостовая пунктуация (`;`, `,`, кавычки) обрезается.
- `:` = порт **только в authority** (до первого `/`); иначе часть пути.
- host извлекается из URL для `host_match`.
- onion-адрес → вариант 1 (спец-TLD, вне обычного PSL).

## Следствия

- Паттерны `.onion`/telegram + правила нормализации — декларативно (`ioc.*`),
  RE2-safe.
- Тест-корпус (case-table + golden) — в `src/test/resources`, синтетический,
  изолированный; выход тестов — в `@TempDir`.
- PSL берём с private-секцией (Guava): duckdns/workers/ngrok/portmap → вариант 1;
  провайдеры вне PSL (`tw1.ru`, …) → вариант 2 — фиксируется в корпусе как
  ожидаемое.
- Текущие юнит-тесты уже изолированы (inline-данные) — соответствуют принципу.

## Открытые вопросы

- Конкретный список «не-PSL провайдеров» в данных — уточняется при наполнении
  корпуса (ожидания на вариант 2).
- Точные base32-границы для `.onion` (v3 = 56 символов) — при реализации паттерна.
