# docs/plan

## Назначение

Каталог содержит детальные планы реализации этапов из
[roadmap.md](../roadmap.md). План фиксирует границы этапа, целевые файлы,
контракты, порядок работ, тесты и критерии приёмки до начала кодовых изменений.

**Правило слоя:** планы не являются источником архитектурной истины. Они
опираются на опубликованный дизайн в `docs/` и не должны подменять
`architecture.md`, `diagnostics.md`, `logging.md` или другие проектные
документы.

## Структура

| Подпапка / файл | Назначение |
|---|---|
| `0006-diagnostics-implementation-plan.md` | План реализации этапа 6: diagnostics |
| `README.md` | Назначение каталога и правила ведения планов |

## Зависимости

**Зависит от:** `docs/roadmap.md`, профильных дизайн-документов этапа,
конвенций проекта.

**Не импортируется:** кодом приложения и тестами; это документация планирования,
а не runtime-контракт.

## Заметки

- Один файл плана соответствует одному нетривиальному этапу roadmap.
- После реализации этапа план может остаться историческим документом, но
  итоговые архитектурные решения переносятся или уточняются в профильных
  документах `docs/`.
